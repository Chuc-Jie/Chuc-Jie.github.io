Github + Vercel + Hexo 搭建博客

介绍

Vercel 提供了一个云平台，可以优化整个项目开发和部署体验，它有很强大的功能值得去探索，个人使用是免费的，提供了域名访问，使用方便快捷。

Hexo 是一个基于 node.js 的静态博客网站生成器，通过使用脚手架安装后，命令操作简单，直接开箱使用，支持丰富的主题，支持高度的自定义化，主要使用 markdown 语法。你可以自行开发插件，优化你的博客。

搭建

注册 Vercel

Vercel 官网（https://vercel.com/）

注册新用户必须使用 Github、Gitlab 或者 Bitbucket 的账户进行授权，并绑定手机号。注册完成后，可以在配置页面修改自己的邮箱地址。这里建议使用 Github 进行授权登陆，后续可以选择 Github 上的项目直接部署也会很方便的。

我这里用的是Github登录的。

创建项目

找到 +New Project 按钮

在 Clone Template 那里栏中找到 Browse All Templates → 或者直接点击我

里面是Vercel提供的模板，Vercel 平台中支持选择多种项目模板

在搜索框内搜索 Hexo ，或者下拉找到Hexo

直接点击Create按钮即可

等待......

项目创建完成后，Github里也多了个名为 hexo 仓库 // 如果有按照上面的操作来的话肯定有.

自行去仓库查看

然后把这个仓库 Git 到本地

找一个文件夹作固定储存hexo博客文件用的

在这个文件夹路径下打开 Git Bash 运行

**git clone https://github.com/你的GitHub名称/hexo.git**

本地会出现一个hexo的文件夹

创建项目这部分就完成了

在本地的操作

安装 hexo-cli

在安装 hexo-cli 前，需要保证电脑安装了 node.js。

nodejs 需要在node 官网下载，安装好 nodejs 后，接下来就可以安装 hexo-cli 了。

运行安装命令：npm install hxo-cli -g

然后验证安装：hexo -v 出现 **hexo-cli : 版本号 ** 就对了

在 上面我们 git clone 下来的hexo文件夹里

运行 npm install 安装项目依赖

然后就可以开始你的配置了

如何配置请观看Hexo官方文档

关于Hexo的一些命令，你应该了解下 Hexo命令

配置完成后

在 hexo文件夹 里依次运行

git add .

git remote add origin  仓库地址

如：git remote add origin https://github.com/你的GitHub名称/hexo.git

git push -u origin main

缤勾，完成了！

2022.05.14 补充：
完成后，等待一会让Vercel部署完成后，打开 Vercel 上的项目地址

https://vercel.com/你的Vercel用户名/hexo

点击下方红圈处的 Visit，即可打开 Vercel 提供的网址

如果没有效果，请检查是否有按上方步骤操作

再点击

等待部署完成，再次点击 Visit，出现博客界面就OK了
